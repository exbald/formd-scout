<project_specification>
  <project_name>FormD Scout</project_name>

  <overview>
    A web dashboard that monitors SEC EDGAR Form D filings to detect companies that have recently raised private funding. Form D filings appear on EDGAR 2-3 weeks before press releases, giving commercial real estate brokers an early signal to identify companies likely to need office space. The app ingests Form D data daily from the free SEC EDGAR API, stores it in PostgreSQL, enriches each filing with AI-generated relevance scoring, and displays everything in a filterable dashboard.
  </overview>

  <sec_edgar_api_reference>
    All SEC EDGAR APIs are free and require NO authentication. The only requirement is a User-Agent header.

    <required_headers>
      All SEC requests MUST include these headers:
      {
        'User-Agent': 'NOCODE-GDN-LLC contact@nocodegdn.com',
        'Accept': 'application/json'
      }
    </required_headers>

    <rate_limit>
      Maximum 10 requests per second to SEC servers. Implement a 150ms delay between requests.
    </rate_limit>

    <endpoint name="EFTS Full-Text Search">
      GET https://efts.sec.gov/LATEST/search-index?q=*&amp;forms=D&amp;dateRange=custom&amp;startdt=YYYY-MM-DD&amp;enddt=YYYY-MM-DD
      Returns JSON with filing metadata. Response structure:
      {
        "hits": {
          "hits": [
            {
              "_source": {
                "entity_name": "Company Name",
                "file_num": "...",
                "period_of_report": "2025-01-15",
                "file_date": "2025-01-15",
                "form_type": "D",
                "file_description": "..."
              },
              "_id": "accession-number"
            }
          ],
          "total": { "value": 100 }
        }
      }
    </endpoint>

    <endpoint name="Company Submissions">
      GET https://data.sec.gov/submissions/CIK{10-digit-cik-with-leading-zeros}.json
      Returns company info and filing history.
    </endpoint>

    <endpoint name="Filing Documents Index">
      GET https://www.sec.gov/Archives/edgar/data/{CIK}/{ACCESSION-NO-WITH-DASHES}/index.json
      Returns JSON listing of all documents in the filing. Look for the primary XML document.
    </endpoint>

    <endpoint name="Form D XML Source">
      GET https://www.sec.gov/Archives/edgar/data/{CIK}/{ACCESSION-NO-WITH-DASHES}/{primary-doc-filename}
      The Form D XML contains structured data:
      - primaryIssuer: Company name, CIK, entity type, address, phone
      - offeringData/industryGroup: Industry classification
      - offeringData/issuerSize/revenueRange: Revenue range
      - offeringData/federalExemptionsExclusions: Exemption types (Rule 506(b), 506(c), etc.)
      - offeringData/typeOfFiling: New filing vs amendment
      - offeringData/dateOfFirstSale: Date or "Yet to Occur"
      - offeringData/investors/totalNumberAlreadyInvested: Investor count
      - offeringData/offeringSalesAmounts: Total offering amount, amount sold, amount remaining
      - offeringData/salesCompensationList: Broker/dealer info
      - offeringData/minimumInvestmentAccepted: Minimum investment amount
    </endpoint>

    <important_notes>
      - The EFTS endpoint response structure may vary. Parse defensively.
      - Form D XML files have inconsistent structures. Never assume a field exists.
      - accessionNumber format uses dashes (e.g., "0001234567-25-000001"). Store with dashes.
      - In URL paths, accession number has dashes REMOVED. Index.json uses dashes.
      - Total offering amounts can be "Indefinite" in XML - store as null.
      - Some filings are for pooled investment funds, not operating companies. Still ingest but they get low relevance scores.
    </important_notes>
  </sec_edgar_api_reference>

      Create src/lib/edgar/parser.ts
      Install fast-xml-parser and parse Form D XML into structured TypeScript objects.
      Function: parseFormDXml(xmlString): ParsedFormDFiling
      Must extract all fields in database schema. Handle missing fields gracefully (return null).
      Handle new filings (form type "D") and amendments ("D/A").
      Parse offering amounts as numbers (remove commas, handle "Indefinite" as null).
      Parse dates properly. Handle "Yet to Occur" for firstSaleDate.
      Must be a pure function with no side effects.
    </form_d_xml_parser>

    6. Some Form D filings are for pooled investment funds, not operating companies. Still ingest but they get low relevance scores.
    7. The AI enrichment prompt must explicitly ask for JSON-only responses. Parse as JSON. If parsing fails, retry once.
    8. All monetary values in UI: >= 1B "$X.XB", >= 1M "$X.XM", >= 1K "$X.XK", below 1K "$X".
    9. Database uses connection pooling via existing db.ts.
    10. Do NOT create cron jobs or schedulers. Ingestion/enrichment endpoints are called externally by n8n.
  </important_implementation_notes>
